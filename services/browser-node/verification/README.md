# Attestation Verification

Verify SEV-SNP attestation reports from the public S3 bucket with full cryptographic validation.

## Quick Start

### Build the Docker image (one time):

```bash
cd services/browser-node/verification
docker build -t attestation-verifier .
```

### Verify any pod:

```bash
docker run --rm attestation-verifier <POD_NAME>
```

**Example:**
```bash
docker run --rm attestation-verifier browser-fleet-9877p-56ddj
```

## What It Verifies

The script performs **three independent security checks**:

### 1. Image Binding Verification
Independently computes the expected Report Data hash and compares against the hardware-signed attestation:

```
Report Data = SHA256( SHA256(imageDigest) || SHA256(cosign.pub) )
```

- **Image Digest** — from the manifest (which container image was attested)
- **Public Key** — embedded in the script from the repo (trusted, versioned)
- **Report Data** — from the hardware-signed attestation report (tamper-proof)

If these match, the attestation is cryptographically bound to **this specific image** and **this specific signing key**.

### 2. VLEK Signature Verification (snpguest)
Validates the AMD certificate chain:

```
ARK (AMD Root Key)
 ↓ Signs
ASK (AMD Signing Key)
 ↓ Signs
VLEK (Versioned Loaded Endorsement Key)
 ↓ Signs
Attestation Report
```

This proves the report was generated by **genuine AMD SNP hardware** and has not been tampered with.

### 3. TCB Version Matching
Verifies that the Trusted Computing Base versions in the attestation match the AMD-signed certificate:
- Boot Loader SVN
- TEE SVN
- SNP Firmware SVN
- Microcode SVN

## Security Design

### Trusted vs Untrusted Sources

| Source | Trust Level | Why |
|--------|------------|-----|
| `attestation.bin` | **Hardware-signed** | Signed by VLEK, chained to AMD root |
| `certs/vlek.der` | **AMD-issued** | Certificate from AMD Key Distribution Service |
| `cosign.pub` | **Repo-versioned** | Embedded in script, same repo as signing key |
| `manifest.json` | Convenience only | Metadata — not relied on for security |

### Why Public Key Is Embedded

The Cosign public key is embedded directly in the verification script rather than fetched at runtime because:
- The script and key live in the **same repo** — they're versioned together
- No network dependency — works offline after S3 download
- Can't be MITM'd — no runtime fetch to intercept
- The key is **public information** — embedding it doesn't reduce security

### Understanding AWS SNP Values

Some values in the attestation report may be zero. **This is normal and secure for AWS:**

- **Chip ID = Zero** — AWS zeros this for privacy/multi-tenancy isolation
- **Some TCB SVNs = Zero** — AWS-managed firmware versioning approach
- **VLEK used instead of VCEK** — Certificate-based, not chip-specific

The VLEK signature validation confirms authenticity regardless of these values.

## Public S3 Access

All attestations are publicly accessible:

```
https://popcorn-attestations-us-east-2.s3.us-east-2.amazonaws.com/{POD_NAME}/
├── manifest.json        # Metadata (convenience only)
├── attestation.bin      # Hardware-signed SEV-SNP report
└── certs/vlek.der       # AMD-issued VLEK certificate
```

## Local Development

Without Docker (no VLEK verification):

```bash
node verify_from_s3.js <POD_NAME>
```

Requires Node.js 20+. VLEK cryptographic verification requires snpguest (included in Docker image).
