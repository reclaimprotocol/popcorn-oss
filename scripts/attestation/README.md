# Popcorn SEV-SNP Attestation Verification

This directory contains standalone scripts to independently verify the cryptographic hardware attestation proofs generated by the Popcorn `attestor` sidecars running inside the AMD SEV-SNP Trusted Execution Environment (TEE).

## How It Works

When a Browser Pod boots up in the Popcorn cluster:
1. It runs on an AMD Epyc processor with SEV-SNP (Secure Encrypted Virtualization - Secure Nested Paging) enabled, ensuring memory encryption and state isolation from the hypervisor and host OS.
2. The `attestor` sidecar container queries the Kubernetes API from within the Pod to retrieve the exact immutable Image Digests (SHA256 hashes) of both the workload (`neko` Chromium browser) and itself.
3. It performs a Cosign verification check on those digests against the trusted Reclaim public key.
4. It calls the `/dev/sev-guest` hardware device to generate an attestation report, passing a SHA-384 hash of the nonces and digests into the 64-byte `REPORT_DATA` field.
5. The hardware returns the report alongside an ECDSA P-384 signature issued by the AMD Versioned Loaded Endorsement Key (VLEK).

The verification scripts in this folder automate the final **Phase 6: Client Verification** step, proving to you (the client) that the remote node is genuinely running the exact signed code you expect.

## Verifying a Session

You can use either the **Go** or **Node.js** implementation. Both perform the exact same comprehensive verification.

### Method 1: Node.js

Requires Node.js installed.

```bash
# Usage:
node scripts/attestation/verify_proof.js <SESSION_ID> [OPTIONAL_NONCE]

# Example:
node scripts/attestation/verify_proof.js session-1x
```

### Method 2: Go

Requires Go 1.20+ installed.

```bash
# Usage:
go run scripts/attestation/verify.go <SESSION_ID> [OPTIONAL_NONCE]

# Example:
go run scripts/attestation/verify.go session-1x
```

### Verification Checks Performed

When you run the script against an active session ID, it will:
1. Fetch the raw proof bundle from the Popcorn Gateway (`/proof/<session_id>`).
2. Read the returned Working Image and Verifier Image digests.
3. Validate that the returned Nonce matches the requested nonce, preventing replay attacks.
4. Recompute the `REPORT_DATA` SHA-384 hash from the digests, public key, and nonce.
5. Decode the AMD SEV-SNP binary report and assert the hardware-locked `REPORT_DATA` perfectly matches the expected hash.
6. Parse the AMD VLEK X.509 Certificate and mathematically evaluate the ECDSA P-384 hardware signature to prove authenticity.

### Artifacts

If verification is successful, the scripts save the raw decoded SEV-SNP binary report and the AMD VLEK certificates to your local `/tmp/proof-<session_id>/` directory:

```
/tmp/proof-session-1x/
├── attestation.bin       # The raw SEV-SNP Guest Report
└── certs/
    └── vlek.der          # The AMD hardware endorsement certificate
```

You can use these artifacts to manually re-verify the signatures using AMD's official tooling if desired.
